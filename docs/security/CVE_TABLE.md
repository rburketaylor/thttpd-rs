### Footnotes

¹ **CWE provenance.** NVD did not assign a primary CWE for these legacy entries
(their record carries `NVD-CWE-noinfo`). The CWE shown is the standard
classification implied by the advisory text and confirmed by the C pattern in
`docs/security/C_PATTERNS.md`. It is marked with ¹ to distinguish it from an
NVD-assigned CWE. We do not invent IDs — each inferred CWE maps to a real, named
weakness in the [CWE catalog](https://cwe.mitre.org/):

- **CVE-1999-1457 → CWE-119** (Improper Restriction of Operations within the
  Bounds of a Memory Buffer): "Buffer overflow … via a long date string, which
  is not properly handled by the `tdate_parse` function."
- **CVE-2002-0733 → CWE-79** (Cross-site Scripting): "Cross-site scripting
  vulnerability … thttpd … inserts the script into a 404 error message."
- **CVE-2004-2628 → CWE-22** (Path Traversal): "Multiple directory traversal
  vulnerabilities … via a URL that contains a hex-encoded backslash dot-dot
  sequence or a drive letter."
- **CVE-2006-1078 → CWE-119** (Buffer Overflow): "Multiple buffer overflows in
  `htpasswd` … via a long command line argument and a long line in a file."
- **CVE-2006-1079 → CWE-78** (OS Command Injection): NVD lists CWE-264; the
  advisory describes "`system` function" shell-metacharacter injection, the
  canonical CWE-78 pattern. Both classes are covered in the mitigation analysis.
- **CVE-2006-4248 → CWE-59** (Link Following): "symlink attack on the
  `start_thttpd` temporary file."

## Reproducibility

The inventory above is regenerated by:

```bash
bash pipeline/refresh_cve_inventory.sh   # re-sweeps OSV + NVD, writes a DRAFT table
```

The script queries NVD CVE API 2.0 for the seed list and OSV for the package
names, then emits a draft with placeholder `Affected` columns. A human fills the
versions from the upstream advisories and commits the curated table (this file).
The `cve_inventory.lock` sidecar records the sweep timestamp and source list for
reproducibility.

### Known sweep limitations

- **OSV package-name queries** for `thttpd`/`sthttpd` return HTTP 400 because
  OSV requires an `ecosystem` field and there is no ecosystem for a C project
  shipped outside a package manager. The NVD seed enrichment path is therefore
  the authoritative source for this family; the OSV pass is retained only to
  catch any future GHSA-linked entries.
- **NVD rate-limits** unauthenticated API requests to ~5/30s. The script sleeps
  6s between requests to stay under the limit; a full 10-CVE sweep takes ~1 min.
- **The Debian security tracker** has no single JSON endpoint, so the
  cross-check that every `src:thttpd` tracker entry has a row here is manual
  (performed during curation on 2026-06-21; all 10 entries map to `thttpd`).
